Network Embedding 🌐

Task: Map each node of a graph into a low-dimensional vector such that:

  1. The embedding can reconstruct the network with high accuracy.
  2. The embedding should be low-dimensional (d \ll N).

Formally, we want \mathbf{U} \in \mathbb{R}^{N \times d} where each row \mathbf{u}_i is the embedding of node i, and nodes with similar roles in the network end up close in \mathbb{R}^d.

Why does this matter? Standard ML algorithms work on vectors. Embedding lets us apply them to graphs.

\underbrace{\mathbf{A}}_{\text{graph}} \;\longrightarrow\; \underbrace{\mathbf{U} \in \mathbb{R}^{N \times d}}_{\text{embedding}}

Q: How would you go about finding such \mathbf{U}?

Idea 1: Network Reconstruction

Compress adjacency matrix \mathbf{A} into \mathbf{U} \in \mathbb{R}^{N \times d} by minimizing the reconstruction error:

\min_{\mathbf{U}} \;\frac{1}{2}\|\mathbf{A} - \mathbf{U}\mathbf{U}^\top\|_F^2

with orthonormal columns: \mathbf{u}_i^\top \mathbf{u}_j = 0 \;(\forall i \neq j) and \|\mathbf{u}_k\|^2 = 1.

For the 1-d case (\mathbf{U} = \lambda^{1/2}\mathbf{u}), this becomes:

\min_{\mathbf{u},\lambda} \; \frac{1}{2}\|\mathbf{A} - \lambda\,\mathbf{u}\mathbf{u}^\top\|_F^2

Setting the gradient \partial/\partial u_i = 0 gives…

Q: Try solving this. Expand the Frobenius norm and differentiate w.r.t. u_i.

Hint: Treat \lambda and all u_j (j \neq i) as constants.

Proof: Gradient → Eigenvalue Problem

Expanding the objective and differentiating w.r.t. u_i:

\frac{\partial}{\partial u_i}\left[\frac{1}{2}\sum_{i,j}(A_{ij} - \lambda u_i u_j)^2\right] = -2\lambda\left[\sum_{j \neq i} A_{ij}u_j - \lambda u_i \sum_{j \neq i}u_j^2\right] - 2\lambda A_{ii}u_i + 2\lambda^2 u_i^3 = 0

Collecting terms (using \sum_j u_j^2 = 1):

\sum_{j=1}^N A_{ij}\,u_j \;=\; \lambda\,u_i \quad\Longrightarrow\quad \boxed{\mathbf{A}\mathbf{u} = \lambda\mathbf{u}}

This is the eigenvalue problem of \mathbf{A}. There are N solutions (\\lambda_k, \mathbf{u}_k). Which eigenvector gives the smallest reconstruction error?

Which Eigenvector to Choose?

Expand the Frobenius norm using \|\mathbf{M}\|_F^2 = \text{Tr}(\mathbf{M}^\top\mathbf{M}):

\tfrac{1}{2}\|\mathbf{A} - \lambda\mathbf{u}\mathbf{u}^\top\|_F^2 = \underbrace{\tfrac{1}{2}\text{Tr}(\mathbf{A}^\top\mathbf{A})}_{\text{const.}} - \lambda\underbrace{\text{Tr}(\mathbf{u}^\top\mathbf{A}\mathbf{u})}_{=\,\lambda} + \tfrac{\lambda^2}{2}\underbrace{\text{Tr}(\mathbf{u}^\top\mathbf{u}\mathbf{u}^\top\mathbf{u})}_{=\,1} = -\lambda^2 + \text{const.}

The error is -\lambda^2 + C, which is minimized when \lambda is the largest eigenvalue.

For d dimensions: the optimal \mathbf{U} is formed from the d eigenvectors of \mathbf{A} corresponding to the top-d eigenvalues.

function buildRecon(d) {
  const Ar = Array.from({length: N}, () => new Float64Array(N));
  for (let k = 0; k < d; k++) {
    const lam = ev_adj[k];
    for (let i = 0; i < N; i++)
      for (let j = 0; j < N; j++)
        Ar[i][j] += lam * U_adj[i][k] * U_adj[j][k];
  }
  return Ar;
}

Idea 2: Graph Cut ✂️

Graph Cut Problem: Partition nodes into two disjoint sets Q and S by cutting the fewest edges:

\min_{Q,S} \;\text{cut}(Q,S) = \sum_{i \in Q}\sum_{j \in S} A_{ij}

Minimizing raw cut is bad — it tends to isolate a single node. We need a balanced cut.

Ratio Cut penalizes imbalanced partitions:

\text{RatioCut}(Q,S) = \sum_{i \in Q}\sum_{j \in S} A_{ij}\left(\frac{1}{|Q|}+\frac{1}{|S|}\right)

Normalized Cut uses volume instead of size:

\text{NC}(Q,S) = \frac{\text{cut}(Q,S)}{\text{vol}(Q)} + \frac{\text{cut}(Q,S)}{\text{vol}(S)}

where \text{vol}(Q) = \sum_{i \in Q} k_i.

Ratio Cut → Laplacian Form

Define the indicator vector: x_i = \begin{cases}\sqrt{|S|/|Q|} & i \in Q \\ -\sqrt{|Q|/|S|} & i \in S\end{cases} \qquad\Rightarrow\qquad \sum_i x_i = 0,\quad \sum_i x_i^2 = N

Note (x_i - x_j)^2 = 0 when i,j are in the same partition, and (x_i-x_j)^2 = N^2/(|Q||S|) otherwise. Then:

\begin{aligned} \text{RatioCut}(Q,S) &= \frac{1}{2N}\sum_{i,j}A_{ij}(x_i-x_j)^2 = \frac{1}{N}\sum_{i,j}L_{ij}\,x_i x_j = \frac{1}{N}\,\mathbf{x}^\top\mathbf{L}\mathbf{x} \end{aligned}

where the combinatorial Laplacian is \mathbf{L} = \mathbf{D} - \mathbf{A}.

So minimizing RatioCut is equivalent to:

\min_{\mathbf{x}}\;\mathbf{x}^\top\mathbf{L}\mathbf{x} \quad\text{s.t.}\quad \mathbf{x}^\top\mathbf{1}=0,\;\mathbf{x}^\top\mathbf{x}=N,\;x_i\in\left\{\sqrt{|S|/|Q|},-\sqrt{|Q|/|S|}\right\}

This is NP-hard. Let’s relax the discrete constraint.

Spectral Relaxation

Relax x_i \in \mathbb{R} with normalization \mathbf{x}^\top\mathbf{x}=1 and \mathbf{x}^\top\mathbf{1}=0:

\min_{\mathbf{x}}\;\mathbf{x}^\top\mathbf{L}\mathbf{x} \quad\text{s.t.}\quad \mathbf{x}^\top\mathbf{x}=1,\;\mathbf{x}\perp\mathbf{1}

From the eigenvalue equation \mathbf{L}\mathbf{x}=\lambda\mathbf{x}:

\mathbf{x}^\top\mathbf{L}\mathbf{x} = \lambda\,\mathbf{x}^\top\mathbf{x} = \lambda

So the minimum is achieved at the smallest eigenvalue… but wait:

Warning

The smallest eigenvalue \lambda_1 = 0 of \mathbf{L} has eigenvector \mathbf{x}_1 \propto \mathbf{1}, which violates \mathbf{x}\perp\mathbf{1}. The second smallest eigenvector \mathbf{x}_2 (the Fiedler vector) is the true solution. For k-way partitioning, use eigenvectors \mathbf{x}_2, \ldots, \mathbf{x}_{k}.

Laplacian Eigenmap 🔄

A closely related embedding: position connected nodes close together in Euclidean space.

Objective: \min_{\mathbf{u}}\;\sum_{i,j}A_{ij}\|\mathbf{u}_i - \mathbf{u}_j\|^2 = \min_{\mathbf{U}}\;\text{tr}(\mathbf{U}^\top\mathbf{L}\mathbf{U}) \quad\text{s.t.}\quad \mathbf{U}^\top\mathbf{D}\mathbf{U} = \mathbf{I}

Solution: The columns of \mathbf{U} are the eigenvectors \mathbf{x}_2, \ldots, \mathbf{x}_{d+1} of the generalized eigenvalue problem:

\mathbf{L}\mathbf{x} = \lambda\,\mathbf{D}\mathbf{x}

The interactive spectral clustering demo above shows these eigenvectors directly. The Laplacian eigenmap is exactly the coordinate system where graph structure becomes geometry.

Neural Embedding: The Key Idea

Spectral methods require eigendecomposition (\mathcal{O}(N^3)). For large networks, we need something faster.

Insight from NLP: word2vec learns word vectors by predicting which words appear near each other in sentences. If two words appear in similar contexts, they get similar vectors.

The translation to graphs:

  1. Run random walks on the graph → sequences of nodes (“sentences”)
  2. Treat node sequences like word sequences
  3. Train word2vec (skip-gram) to predict which nodes co-occur in walks

\mathcal{L} = \sum_t \sum_{0 < |j| \le c} \log P(v_{t+j} \mid v_t), \qquad P(v_O \mid v_I) = \frac{\exp(\mathbf{u}_{v_O}^\top \mathbf{u}_{v_I})}{\sum_v \exp(\mathbf{u}_v^\top \mathbf{u}_{v_I})}

DeepWalk (Perozzi et al., 2014) does exactly this with uniform random walks.

node2vec: Biased Random Walks

node2vec (Grover & Leskovec, 2016) extends DeepWalk with a biased walk controlled by two parameters:

P(\text{next}=x \mid v, t) \propto \begin{cases} 1/p & d(t,x) = 0 \text{ (backtrack to } t)\\ 1 & d(t,x) = 1 \text{ (shared neighbor)}\\ 1/q & d(t,x) = 2 \text{ (explore further)} \end{cases}

  • Low p: frequent backtracking → BFS-like, explores local neighborhood → captures structural equivalence
  • Low q: frequent exploration → DFS-like, ventures far → captures community membership

The walk at node v (came from t) assigns unnormalized weights to neighbors x:

x == t      →  weight 1/p
x ~ t       →  weight 1
otherwise   →  weight 1/q

Then normalize and sample. The walk is second-order Markov (depends on both current and previous node).

adj_list = {
  const al = Array.from({length: N}, () => []);
  for (const e of edges_js) { al[e.s].push(e.t); al[e.t].push(e.s); }
  return al;
}

// ── Biased random walk ────────────────────────────────
walk = {
  const seed_val = rw_start * 31 + rw_steps;  // pseudo-seed for stability
  const rng = (() => { let s = seed_val; return () => { s = (s * 1664525 + 1013904223) & 0xffffffff; return (s >>> 0) / 4294967296; }; })();
  const w = [rw_start];
  let prev = null;
  for (let step = 0; step < rw_steps - 1; step++) {
    const cur   = w[w.length - 1];
    const nbrs  = adj_list[cur];
    if (!nbrs.length) break;
    if (prev === null) {
      w.push(nbrs[Math.floor(rng() * nbrs.length)]);
    } else {
      const wts = nbrs.map(x => {
        if (x === prev) return 1 / rw_p;
        if (A_mat[prev][x] > 0) return 1.0;
        return 1 / rw_q;
      });
      const total = wts.reduce((a,b)=>a+b, 0);
      let r = rng() * total, chosen = nbrs[nbrs.length-1];
      for (let i = 0; i < nbrs.length; i++) { r -= wts[i]; if (r <= 0) { chosen = nbrs[i]; break; } }
      w.push(chosen);
    }
    prev = cur;
  }
  return w;
}
